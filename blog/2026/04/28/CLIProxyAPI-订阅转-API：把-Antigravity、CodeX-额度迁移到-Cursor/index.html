CLIProxyAPI 订阅转 API：把 Antigravity、CodeX 额度迁移到 Cursor

现在，大模型能力越来越强，但 API 价格也越来越贵。如何把已有的订阅额度“用到极致”，成了一个很现实的问题。

老板开通 Codex 后，总抱怨： 在 CodeX 的 Pro 生成计划几乎不怎么消耗额度；但同样的事情放到 Cursor 里执行，动不动就是 20%～30% 的消耗。

于是，很自然的萌生了一个想法： 能不能把 CodeX 的订阅Token，包装成接口接到 Cursor 里面用呢？ 同时，转成 API 后，也可以在自己的代码中使用了。

毕竟，老板对 Cursor 情有独钟，尤其认可它在多 Agent 协作、系统提示词等方面的优化。

# 方案探索

这种需求肯定不只我们有，毕竟 API 价格比订阅贵不少。

搜了一圈（Google + AI），还真找到了同样需求的文章：CLIProxyAPI 把 Claude Code、Gemini CLI、Codex 订阅包装成统一 API 的开源神器 一个代理层，撬动所有 AI CLI 订阅的 API 能力。

https://blog.einverne.info/post/2026/04/cliproxyapi-unified-ai-gateway-for-cli-subscriptions.html

以及文章中提到的功能完整的开源方案：CLIProxyAPI。

👉 一套统一 AI 网关，可以把各种订阅型服务转成标准 API 使用。

项目介绍：https://github.com/router-for-me/CLIProxyAPI#getting\-started

# 快速上手

项目是用 Go 写的，Windows 直接下载编译好的二进制文件：CLIProxyAPI_6.9.40_windows_amd64.zip。

下载地址： https://github.com/router-for-me/CLIProxyAPI/releases/tag/v6.9.40

# 1. 修改配置

先不用细究每个配置，直接改最关键的，运行起来后再慢慢体会：

host: "127.0.0.1"

一开始搞清楚两个key就可以了：

•management-key：用于登录管理后台•dev-key：作为调用大模型 API key

# 2. 启动服务

直接双击运行：

cli-proxy-api.exe

# 3. 登录管理界面

参考文档： https://help.router-for.me/management/webui.html

登录后，在后台添加你的订阅服务 Token（比如 CodeX、Antigravity 等）。

# 4. 命令行授权（可选）

如果你更习惯 CLI，也可以通过命令行方式授权登录：

https://help.router-for.me/configuration/provider/antigravity.html

# 简单测试

服务启动后，可以用 curl 做一个最小验证：

curl http://127.0.0.1:8317/v1/chat/completions \

如果返回正常，就说明整个链路已经打通。

# 接入 Agent / 客户端

参考官方的 Codex 接入方式： https://help.router-for.me/agent-client/codex

如果默认模型列表里没有你要用的模型，可以：

•点击 View All Models•滚动到底部•选择 Add Custom Model

手动补充模型定义即可。

# 实测情况

我这边是用 Google Antigravity 的订阅来登录测试的：

❌ GPT：不包含的 OpenAI 模型不可用

✅ Gemini：提供了的 Google 模型可以正常调用

整体链路是通的，代理方案整体也很顺畅。

# 小结

这个方案的核心价值：

•把“订阅额度”转成“统一 API”•再接入像 Cursor 这样的工具使用•达到 降低成本 + 最佳体验 的效果

接下来，就可以丢给老板去配，看看他订阅的 CodeX 大模型在 Cursor 里面“哒嗯哒”了 😄
