微信 AI 与 WebMCP：AI Agent 时代，正在把应用变成 AI 的工具

最近体验了一下微信小程序 AI 开发模式，第一反应是：它和前阵子 Chrome 推出的 WebMCP 非常像。

一个面向 Web，一个面向微信小程序，但它们似乎都在做同一件事情：让应用不再只是一个给人操作的界面，而是主动向 AI Agent 暴露自己的能力。

这其实也是 AI Agent 时代正在发生的一次变化。

过去的软件交互模式是：

用户 → 浏览器 / App → 网站 / 小程序。

用户需要理解软件，知道入口在哪里，知道下一步应该点击什么。

而未来可能会变成：

用户 → AI Agent → 应用能力 → 业务系统。

AI 不需要像人一样去理解页面、寻找按钮、模拟点击，而应直接获取应用有哪些能力，需要什么参数，以及如何执行。

# WebMCP：让网页成为 AI 可以调用的工具

第一次看到 WebMCP 时，感觉这个方向非常有意思。

WebMCP 是 Chrome 面向 AI Agent 探索的一套 Web 标准，让网页主动暴露能力  Tool，让 AI 可以直接发现并调用。https://developer.chrome.com/docs/ai/webmcp?hl=zh-cn#use\_webmcp\_apis

例如，一个酒店网站。

以前 AI 如果想帮用户订酒店，需要先理解页面结构：

搜索框在哪里？

而有了 WebMCP 后，网站可以直接告诉 AI：

我提供一个 searchHotel 工具，需要 city、checkInDate、days 这些参数。

AI 不需要猜页面结构，也不需要模拟鼠标操作，而是直接调用这个能力。

网页从“展示信息的页面”，变成了“可以被 AI 使用的工具”。

# 跑一下 WebMCP Demo

Google Chrome 提供了 Model Context Tool Inspector，可以直接体验 WebMCP。

打开官方 Demo：

https://googlechromelabs.github.io/webmcp-tools/demos/react-flightsearch/

然后，在 Inspector 的 User Prompt 中输入：

“帮我查 JFK 到 LAX，2026-06-20，单程，1 人，只显示直飞航班。”

它会自动发现页面提供的 tools，例如：searchFlights、setFilters、listFlights。随后 AI 直接调用这些工具完成任务。

整个过程不需要额外写桥接代码，也不需要分析 DOM，更不需要通过自动化脚本模拟点击。

原因很简单：页面已经主动告诉 AI 自己有什么能力。

官方版本 Model Context Tool Inspector 只支持 Gemini。

如果使用其他模型，可以用 Cursor 改一版 Inspector，把模型调用部分替换成自己常用的大模型。

修改之后，Tool 的发现、参数解析、调用流程都不需要改变。

# 微信 AI：小程序也开始 Agent 化

再来看最近微信推出的小程序 AI 能力。体验之后发现，它和 WebMCP 的思路非常接近。

以前的小程序：

用户打开页面 - 寻找入口 - 点击按钮 - 完成流程。

未来用户可能只需要告诉 AI：

“帮我查一下订单。”

AI 根据小程序提供的 Skill，找到对应能力，然后调用完成任务。小程序也从 UI 驱动，逐渐走向能力驱动。增加了原子接口、原子组件，对应 MCP 工具的实现及其结果呈现。

接下来，先把官方的 Demo 跑起来。 

先来配置小程序开发环境，参考 https://developers.weixin.qq.com/miniprogram/dev/ai/guide.html 。

1、安装最新版的小程序开发者工具。

2、开通AI能力的开发模式。

3、下载 https://github.com/wechat-miniprogram/ai-mode-demo 到本地，再导入demo小程序，然后选择 AppID 开通了开发模式的小程序。

4、选择 小程序 AI 编译，试试对话：

# 旧小程序如何升级？

微信提供了对应的升级路径。

开发者可以基于已有的小程序页面，提取功能，生成 AI Skill，并适配 AI 原子组件，把已有能力开放给 AI。

实际测试中，也发现它的一些限制，例如不能随意读取小程序内部文件，复杂业务逻辑仍然需要开发者主动封装。

但方向已经很明确：

未来的小程序，不只是给用户点击使用，也需要让 AI 理解和调用。

# 从 WebMCP 到微信 AI：应用正在变成 AI 的基础设施

过去几十年，软件开发主要围绕 UI：页面、按钮、表单、交互流程。

开发者设计的是：“人如何使用软件。”

但 AI Agent 出现之后，应用需要增加另一层设计：能力、工具、参数、调用方式。

以前 API 是给程序调用的。

未来 Skill / Tool 是给 AI 调用的。从 WebMCP 到微信 AI，可以看到一个共同趋势：应用正在从“面向人的界面”，变成“人和 AI 都可以使用的能力入口”。

这可能会成为未来几年应用开发最大的变化之一。
