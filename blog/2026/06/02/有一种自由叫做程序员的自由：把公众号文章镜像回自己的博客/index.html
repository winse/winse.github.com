有一种自由叫做程序员的自由：把公众号文章镜像回自己的博客

最近，折腾优化了一下自己的网络环境，连着写了几篇相关的文章。结果却让人很郁闷，分享的这几篇实战文章，都被平台封掉了。

这些文章本来就不是追热点，更不为博流量，只是记录自己日常折腾和实践过程的技术笔记。很多时刻，写下来不仅是给大家看，更是给未来的自己留作一份参考。结果现在文章被这么一封，连我自己回头查资料想看都看不到自己的文章了。

说实话，这就让我很纠结，甚至有那么点出离的愤怒。

更无奈的是，即使用付费的方式也绕不过去🥹。

既然不然发，那就换一种更纯粹的、更自由的方式来储存自己的记忆。兜兜转转，又回到最原始的老方案：用 Markdown 的格式来存，用 Github Pages 静态站点的呈现。内容掌握在自己手里，不依赖任何平台，也不用担心突然消失。

但是，有一个现实问题：近一年多的文章只发布在公众号上面。我得把公众号的历史文章和草稿箱全量地导出来。

搜了一圈后，发现已经有不少类似的开源小工具。于是花了几天的功夫，借助 Cursor Agents Window，让 AI 参考现有源码的思路，做成一个能导出公众号文章的 Skill：不仅能导出公众号文章为 Markdown，文章中图片、封面、摘要等内容也一并导出保存下来。

以前是直接用 Notepad 写 Markdown，比较随性随意。而在公众号上写文章，自己反而会不自觉的要求严格、严谨一些。

所以，后面文章依然在公众号上创作和沉淀，但写好之后，再用 Cursor Skill：wechat-mp-to-11ty 将草稿箱内容导出为 Markdown，以后就同时发布到公众号和个人博客，两个地方同步双保险。

平台有平台的价值，但自己的内容，还是得掌握在自己的手里。

下面这图是很多年前 Jekyll + Octopress 搭建的博客页面。

这一次也进行一下升级，让 AI 帮我改版，把原来的 Ruby 迁移到 Nodejs 生态，使用非常棒的静态站点生成器 Eleventy（ https://github.com/11ty/eleventy/ ）。

把散落的文字，又一篇篇的重新存放到属于自己的地方，技术的开始、后来持续记录点滴，积淀、实践与折腾，轨迹与脉络。回头翻看，不知不觉，好快啊一下就十几年了。
