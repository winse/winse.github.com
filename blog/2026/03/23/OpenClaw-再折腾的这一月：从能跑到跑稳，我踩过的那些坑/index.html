OpenClaw 再折腾的这一月：从能跑到跑稳，我踩过的那些坑

这一个半月把 OpenClaw 跑在树莓派上，从最开始能跑就行，到现在基本稳定，可能出现的问题心中有数，中间踩了不少坑，也慢慢有了一点自己的使用节奏。

# 关于更新

这个月内升级了几次，每次都挺谨慎的，甚至可以说战战兢兢。

OpenClaw 现在还在快速迭代，有些更新不是简单修 bug，而是结构大变，比如飞书从社区插件变成了内置能力，这种改动如果还想着在原有的程序上升级，往往会越搞越乱。

后来我就不纠结了：遇到这种大改动，直接重装最省事。

把 workspace 配置留着，其它按最新的文档重新配一遍，反而更干净。现在，官方文档确实比之前好很多，基本能跟着走下来。

# 关于日常维护

日常最常遇到的还是配置问题。OpenClaw 自带的 openclaw doctor --fix 挺好用，出问题先跑一遍，很多环境问题都能直接修复掉。

但有一个坑踩了好几次：doctor 修完之后，会顺手把 VPN 代理也加上，结果服务是起来了，但飞书直接连不上。刚开始还以为是权限问题，后来才发现是代理导致的。

处理也简单，把代理删掉，再重启就好了。

# 关于飞书的流输出

飞书流式输出我配了，官方文档里写得挺清楚的。

https://docs.openclaw.ai/channels/feishu#streaming

还需要按文档完善权限，加上就能用了。

https://docs.openclaw.ai/channels/feishu#4\-configure-permissions

但实际用下来没太大感觉，我一般也不会盯着看实时的输出，都是等飞书有消息提醒了再看结果。

所以这个功能对我来说属于可有可无，不加也不影响使用。

# 关于用电

一开始也算过功耗，毕竟是 24 小时跑着的。

https://docs.openclaw.ai/install/raspberry-pi#prerequisites

但实际测下来，还是有点过度焦虑了，树莓派本来就低功耗，这点电基本可以放心跑。日常随便忘关一晚的灯，可能都比它跑半月还耗电。

所以这块后来就不折腾了，稳定跑着就行。

当然，加个智能开关 还是很有必要的，管理起来就非常的便利了。偶尔要是发消息后没反应，机器假死，直接远程开关一按一重启就行了。

# 关于模型

模型前面试过字节、MiniMax 体验版，几块钱的用着还行。

后来优惠没了，就换到了阿里云的 Coding Plan，交个朋友 7.9，后面又续了一个月的半价，用它来跑 OpenClaw 完全够用。

这块的感觉是，大模型的价格都很透明，大差不差，能稳定用、没必要换来换去。
