笔记本电脑扩展出多个屏幕：Sunshine + Moonlight 实战记录

上一篇分享了我折腾的三种笔记本扩展屏幕方案笔记本屏幕不够用？那就再加一块：三种简单副屏方案（Windows / 手机 / 平板）。没想到评论区直接给我补了一课——有人提到 Sunshine + Moonlight 这套组合。

一开始没太在意，一试之后发现：这东西有点强。

•✅ 跨平台（Windows / Linux / Android / iOS）•✅ 开源免费•✅ 支持多屏扩展•✅ 延迟低、体验接近原生副屏

所以这篇就把完整折腾过程记录下来，以备不时之需，也希望对你有帮助。

# 一、核心思路：太阳 + 月亮 + 镜子 🌞🌙🪞

这套方案可以理解成一句话：

👉 Sunshine 负责“发光”，Moonlight 负责“接收”，虚拟显示器负责“造屏幕”

用到三个核心工具：

•Sunshine（电脑端服务）•Moonlight（副屏设备）•虚拟显示器（可选，主要用于多屏扩展）

相关地址：

• Sunshine https://github.com/LizardByte/Sunshine

• Moonlight（Android） https://github.com/moonlight-stream/moonlight-android

• 虚拟显示器驱动（parsec-vdd） https://github.com/nomi-san/parsec-vdd

# 二、Sunshine 配置（主机端）

# 1. 启动服务

下载并解压 Sunshine，直接运行：

启动后，在任务栏会看到图标  。

在图标上右键 → Open Sunshine，进入配置页面。

# 2. 初始化登录

第一次打开需要设置一个管理密码。

设置完之后登录后台。

# 3. 屏幕配置（关键步骤）

默认配置其实已经可以用来“复制屏幕”，但我们要的是：

👉 扩展屏幕

这里有两种增加虚拟显示器的方式：

# 方式一：使用物理虚拟显示器（最简单）

在 Sunshine 的输出配置中：

1.打开黑窗口（Sunshine 启动日志）2.查找显示器的 UUID

3.填入配置

4.填写好显示器的 uuid 后，保存并应用。

✔ 不需要重启，立即生效

# 方式二：使用软件虚拟显示器（进阶）

如果你没有虚拟显示器卡，就用：

👉 parsec-vdd 虚拟显示器

安装后：

1. 安装驱动 parsec-vdd

2. 打开 ParsecVDisplay 控制面板

3. 创建虚拟显示器

4. 在 Sunshine 中配置这个虚拟屏（在命令行中查看虚拟显示屏的uuid）

这样就“凭空造”出一块屏幕。

# 三、Moonlight 连接（副屏设备）

在手机 / 平板上安装 Moonlight。

打开后：

1.自动扫描局域网主机2.选择你的电脑

3.点击连接

第一次连接时：

•电脑会弹出提示，点击它弹出配对页面

•输入手机上显示的 PIN 码•确认服务名称（默认 Sunshine）

✔ 完成后，这台设备就变成了一块“屏幕”

# 四、多屏扩展（重点玩法）

如果你只做到这里，其实只是“一块副屏”。

真正有意思的是：

👉 多开 Sunshine，实现多块屏幕扩展

# 1. 启动多个 Sunshine 实例

再启动一个 Sunshine 服务，用不同端口和配置文件。

# 2. 配置不同实例

每个实例：

•设置不同名称

•绑定不同显示器（物理 or 虚拟）

例如：

•Sunshine-1 → 硬件虚拟显示器 A•Sunshine-2 → 软件虚拟屏 B

# 3. 多设备连接

然后：

•手机 1 → 连接 Sunshine-1•手机 2 → 连接 Sunshine-2

👉 最终效果：

你的电脑可以同时扩展出：

•平板屏•手机屏•虚拟屏

甚至三屏、四屏都没问题。

# 五、实际体验

整体体验总结一下：

# 优点

•延迟低（局域网下几乎无感）•画质可调（带宽换清晰度）•支持多屏扩展（这是最大亮点）•开源，可控性强

# 注意点

•网络质量很关键（建议 5GHz WiFi / 有线）•虚拟显示器需要额外配置•多实例时端口和设备绑定要理清

# 六、总结

如果你只是想要一个副屏，很多方案都能做到。

但如果你想：

👉 低成本 + 多设备 + 灵活扩展

那这套：

Sunshine + Moonlight + 虚拟显示器

确实是目前我用下来最自由的一种方案。
